# Граница продуктов: WB_Repricer vs WB_Profit_Ready

## WB_Repricer (эта папка) — только контроль цен на WB

- Токен **продавца** (discounts-prices API): каталог, текущие цены/скидки в кабинете.
- **Парсинг витрины** (Playwright, сессия покупателя): цена «WB Кошелёк», обычная цена на карточке; учёт **региона** (`dest`) для отображаемых цен и СПП.
- **Целевая цена** по `nmId`, снимки мониторинга, отклонения.
- **Удержание цены**: джоб сравнивает цену кошелька с целью и при необходимости отправляет **новую цену до скидки** в кабинет через `POST .../api/v2/upload/task` (ограничение шага за проход, защита от резкого снижения → карантин WB).

Здесь **нет**: учёта прибыли, себестоимости, матрицы маржи, отчётов по продажам, Python-стека Price Guard как основного приложения.

## WB_Profit_Ready (отдельный проект)

- Юнит-экономика, прибыль, Express/статистика продавца и связанный веб-функционал.
- Может содержать **копию** wallet CLI; каноничная версия парсера кошелька для repricer — в `WB_Repricer/src/walletDom/`.

## Практика

1. Разработка **repricing + DOM кошелька** — только в **WB_Repricer**.
2. Изменения в **прибыли/матрице** — только в **WB_Profit_Ready** (или другом репозитории), без смешивания бизнес-логики в repricer.
3. Деплой на сервер: один процесс `node dist/server.js` из **WB_Repricer**, отдельный systemd/docker unit; не путать с сервисами Profit.
